Test Case FINISH-0020
Title Pressing back while podcast is playing, terminates play
Objective

This test case is intended to verify the following implied or specified requirements:

  • A podcast shall be "completed" if the user presses the "back" navigation control returning to the main menu.

This test case demonstrates that while playing a podcast, if a user presses the back, the podcast is terminated and it will be treated as though the podcast has completed.
Setup
  • Install the app into an Android phone
  • Install podcasts into the podcast directory
Actions
  1. From the main menu, select a podcast.
  2. Confirm the podcast begins playing and the play menu is displayed.
  3. Press the back button.
  4. Verify the audio play stops.
  5. Select the same podcast.
  6. Verify that the podcast restarts play at the beginning of the audio.
Expected Pressing the back kill terminates the play of the audio currently in progress and treats this the same as playing the podcast to the end. That is, if it is selected again, play starts at the beginning.
Results PASSED
Cleanup
  • Unnecessary
