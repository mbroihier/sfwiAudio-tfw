Test Case STARTUP-0030
Title Application started with file permissions but no podcasts
Objective

This test case is intended to verify the following implied or specified requirements:

  • When the app is started by selecting from the Android app selection menu, the app shall do the following:
  • If there are no podcasts in the directory, the app shall display a message stating that the device is empty.
  • The app shall require no special provisioning however the podcast settings may be required to be adjusted by the Android OS.

This test case illustrates that an error message will be displayed when permissions are given to access local files, but there are no files to display.
Setup
  • Application is installed
  • Local file access is enabled for the application
  • No files are in the podcast directory
Actions
  1. Start the application.
  2. Verify a message is displayed indicating that there are no podcasts in the podcast directory.
Expected An error message is issued warning the user that no podcasts are available.
Results PASSED
Cleanup
  • Unnecessary
Requirements SFWIAUDIO-PR-INIT-010, SFWIAUDIO-PR-INIT-010.001, SFWIAUDIO-PROV-010
