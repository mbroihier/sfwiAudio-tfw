Test Case STARTUP-0010
Title Application start with no podcasts in the podcasts directory
Objective

This test case is intended to verify the following implied or specified requirements:

  • When the app is started by selecting from the Android app selection menu, the app shall do the following:
  • If there are no podcasts in the directory, the app shall display a message stating that the device is empty.
  • The app shall require no special provisioning however the podcast settings may be required to be adjusted by the Android OS.

By running the application with no podcasts in the podcast directory, this test case illustrates that the app is stable with no podcasts to play and that it notifies the user that the directory is empty.
Setup
  • Application is install on an Android phone
  • Podcast directory of the phone is empty
Actions
  1. Start the application.
  2. Verify that a warning message is displayed and record the message in the test log.
Expected The application displays a warning message.
Results UNTESTED
Cleanup
  • Unnecessary
Requirements SFWIAUDIO-PR-INIT-010, SFWIAUDIO-PR-INIT-010.001, SFWIAUDIO-PROV-010
