Test Case PERFORMANCE-0010
Title App displays and can play all podcasts in the podcasts directory
Objective

This test case is intended to verify the following implied or specified requirements:

  • The app shall be able to display as many podcasts as there are in the directory.
  • The app shall play the podcast a normal speed.

When the user starts the app, all of the podcasts in the podcast directory of the phone should be displayed and they should be available to play. This test case illustrates that and verifies that the audio sounds normal.
Setup
  • App is installed on Android phone
Actions
  1. Install 5 podcasts into the podcast directory of the phone.
  2. Start the app.
  3. Verify that five podcasts are shown in the main menu list.
  4. Select the first podcast in the list.
  5. Verify the podcast plays and it is the correct one.
  6. Press the back key.
  7. Verify the podcast terminates and that the main menu is presented.
  8. Select the next podcast and repeat the above procedure verifying that all podcasts are playable.
Expected Five podcasts will be displayed and all five will be available for play and will sound normal.
Results FAILED
Cleanup
  • Unnecessary
Requirements SFWIAUDIO-PERF-010, SFWIAUDIO-PERF-020
